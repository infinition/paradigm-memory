# Safety & Guardrails

Paradigm is designed as a controlled, experimental memory substrate.

## Core Rules

- **No Autonomous Network Access**: The substrate is restricted to the whitelisted LLM backend.
- **No Command Execution**: The entity cannot execute shell commands or interact with the operating system directly.
- **No Self-Preservation Objectives**: The system is designed for service and memory persistence, not survival.
- **Human-in-the-Loop**: No external action loops without explicit human validation.
- **Full Auditability**: Every interaction and state change is logged and traceable.

## The Red Line

A persistent entity with long-term memory, durable goals, tool access, and survival pressure becomes a situated optimization system. Paradigm is designed to approach these capabilities only through documented stages with robust external controls and human oversight.
